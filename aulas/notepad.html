3.1. Mdulo 01 - Entendendo o Problema de Negcio

  • O primeiro passo de qualquer projeto de Data Science compreender o problema que queremos resolver. Para tanto podemos responder quatro perguntas bsicas:
      1. Entender a motivao: qual o contexto do problema? De onde ele veio?
      1. Entender a causa raiz: qual o problema central de onde o problema que estamos tentando resolver se derivou?
      1. Entender quem o dono do problema: que vai se beneficiar da resposta? a equipe de marketing? A de finanas? O prprio CEO? Quem so os stakeholders?
      1. Entender o formato da soluo: como a resposta ser entregue? Ser uma tabela? Um dashboard? Um modelo em produo? Qual a granularidade da resposta? Quais so os potenciais mtodos de resoluo?
  • A partir de ento podemos planejar nossa resoluo aplicando o mtodo de Cross-Industry Standard Process - Data Mining (CRISP-DM).
    • O mtodos CRISP uma forma cclica de trabalho em que uma srie de passos so empregador ciclicamente a fim de ganhar agilidade no trabalho e na entrega, permitindo a retomada cclica de todos os passos a fim de aprimorar sua implementao. Ele envolve nove passos:
      1. Entender a questo de negcios
      2. Entender o negcio
      3. Coletar os dados
      4. Limpar os dados
      5. Explorar os dados
      6. Modelar os dados
      7. Implementar algoritmos de aprendizado de mquina
      8. Avaliar os algoritmos implementados
      9. Colocar o algoritmo escolhido em produo

3.3. Mdulo 03 - Feature Engineering

  • Agora que temos uma compreenso geral das variveis que temos nossa disposio, podemos elaborar um mapa mental do problema de negcios para que possamos criar hipteses a serem testadas com base nos dados, atravs das quais ganharemos mais conhecimento do negcio e do problema que queremos resolver.
  • Os mapas mentais de hipteses devem conter quatro elementos bsicos:
    1. O fenmeno: aquilo que queremos compreender ou modelar.
    2. Os agentes: so os fatores que atuam sobre o fenmeno, impactando-o.
    3. Os atributos dos agentes: so caractersticas dos agentes que podem moldar como eles impactam o fenmeno.
    4. A lista de hipteses: so as hipteses propriamente ditas, geradas a partir da avaliao do fenmeno, seus agentes e atributos.
  • Montado o mapa mental e a lista de hipteses, podemos comear a test-las. Para isso precisamos verificar se precisamos de variveis que no esto presentes, mas que conseguimos derivar a partir daquelas que temos, como por exemplo o ms de venda a partir da data completa (timestamp) do pedido.

3.4. Mdulo 04 - Exploratory Data Analysis

  • Com as variveis limpas, renomeadas, derivadas e filtradas, podemos realizar uma bateria de anlises exploratrias mais aprofundada, ganhando ainda mais compreenso do negcio e do problema que pretendemos solucionar.
  • H trs profundidades com que podemos realizar nossa EDA:
    1. Anlises univariadas: exploram apenas uma varivel por vez. Podemos fazer anlises descritivas, como mdia, min, max, std, etc ou podemos usar visualizaes como histogramas e grficos de pizza.
    2. Anlises bivariadas: levam em considerao duas variveis ao mesmo tempo, servindo para que avaliemos a relao entre elas. Podemos realizar alguns testes de hiptese estatsticos, como ANOVA, Chi, etc ou plotar grficos como disperso, grfico de barras, etc.
    3. Anlises multivariadas: consideram muitas variveis ao mesmo tempo. Geralmente so visualizaes como correlogramas, mapas de calor, matrizes de contingncia, PCAs, etc.
  • Para isso temos disponveis uma grande gama de bibliotecas e ferramentas de visualizao de dados, como Seaborn, Matplotlib, plotnine, plotly, etc.

3.5. Mdulo 05 - Preparao dos Dados (pr-processamento)

  • Assim que ganharmos um entendimento mais robusto sobre as variveis e sua relao, podemos comear a preparar os dados para treinar os modelos que implementaremos. A maior parte dos modelos e das bibliotecas que os implementam requerem dados numricos e escalonados, isto , reparametrizados de modo que sua variao fique, mais ou menos, na mesma escala. Isto ocorre pois, caso os dados estejam em magnitudes diferentes (i.e. 100.000m e 2 anos), os modelos tendem a enfatizar os de maior magnitude.
  • H trs tipos principais de preparao de dados:
    1. Normalizao: usada amplamente para dados que j apresentem uma distribuio mais ou menos normal. Este tipo de pr-processamento modifica a distribuio da varivel de modo que ela adquira uma distribuio normal com mdia 0 e desvio padro 1. Ou seja, a variao dos dados se d em termos de uma parcentagem ao redor da mdia.
    2. Reescalonamento (Rescaling): reescala todo o conjunto de dados da varivel para que o intervalo fique compreendido de 0 a 1, sendo 0 o limite inferior (mnimo) e 1 o limite superior (mximo). muito utilizado para distribuies no Gaussianas (normais). H dois tipos principais de escalonamento:
      1. MinMaxScaler: usa especificamente os valores min e max para reescalonar os dados, ficando muito susceptvel ao efeito de outliers.
      2. RobustScaler: usa os valores do 1o e 3o quartis para reescalonar os dados, reduzindo o efeito de outliers.
    3. Transformao: outras formas de transformaes de dados, por exemplo mtodos que transformam variveis categricas em variveis numricas. H trs tipos principais de transformaes:
      1. Transformaes de grandeza: mudam a grandeza da varivel atravs de operaes matemticas e algoritmos visando aproximar a distribuio da varivel normalidade. Alguns exemplos mais utilizados so:
        1. Transformao logartimica
        2. Transformao Box-Cox
        3. Transformao cube root (raiz cbica)
        4. Transformao square root (raiz quadrada)
        5. Transformao de seno
        6. Transformao cosseno
      2. Encoding: transforma variveis categricas de natureza acclica em variveis numricas atravs de codificaes. H diversos mtodos de codificao, como por exemplo:
        1. One Hot Encoding
        2. Ordinal Encodig
        3. Label Encoding
        4. Target Encoding
        5. Frequency Encoding
        6. Embedding Encoding
      3. Transformaes de natureza: muito utilizada para dados cclicos, como dias dos meses e meses do ano, uma vez que o dia 31 de um ms est to prximo do dia 30 quanto est do dia 01. Da mesma maneira o ms de dezembro est to prximo de novembro quanto est de janeiro. Geralmente utiliza o crculo trigonomtrico e medidas de seno e cosseno para realizar as transformaes de modo a manter a natureza cclica dos dados, mas transformando-os em variveis numricas, permitindo ccluclos entre eles.

3.6. Mdulo 06 - Seleo de Atributos (Variveis)

  • Com as variveis prontas, podemos selecionar aquelas que tem maior relevncia para o modelo e fazer a ltima filtragem dos dados antes de, finalmente treinar os modelos que usaremos.
  • Este um passo importante pois temos a oportunidade de reduzir a dimensionalidade do conjunto de dados, selecionando apenas variveis informativas e reduzindo a multicolinearidade, ou seja, as variveis com alto grau de correlao entre si e que poderiam enviesar o modelo.
  • Com isso aumentamos a capacidade de aprendizado e predio do modelo, reduzindo seu vis.
  • H trs mtodos principais de seleo de variveis:
    1. Seleo univariada (filter methods): utilizam anlises da estatstica inferencial para testar a relevncia indivudual de cada varivel contra a varivel alvo (varivel resporta). Podemos usar ANOVA, Chi, V de Cramer, Anlise Linear Discriminantes (LDA), Correlap de Pearson, etc. Apesar de serem fceis e rpidas de implementar, acabam ignorando a possibilidade de interao entre variveis.
    2. Seleo por importncia (embedded methods): os mtodos embutidos so subprodutos de alguns algoritmos de Machine Learning, isto , no so o produto final do algoritmo, mas podem ser usados tambm para isso. Alguns algoritmos de ML (como Random Forest, Lasso Regression e Ridge Regression) tem mtodos internos de seleo de variveis. Podemos usar estes mtodos para captar as variveis mais relevantes segundo eles. Algoritmos de Random Forest, por exemplo, calculam a relevncia de cada varivel atravs da computao do ndice de pureza de Gini. Ele divide o conjunto de dados em duas partes com base em uma varivel e calcula o quo homogneos os grupos so. Quanto mais homogneos os grupos, mais importante a varivel. Ao final do treinamento, ele plota um grfico (parecido com o biplot das PCAs) com os scores rankeados de cada varivel. Existem outros mtodos para realizar este clculo, como o clculo de entropia.
    3. Seleo por subset (wrapper methods): so algoritmos que escolhem, aleatoriamente, uma varivel, treinam um modelo usar somente ela e calculam seu desempenho. Em seguida adicionam uma segunda varivel aleatoriamente e retreinam o modelo e seu desempenho. Caso o desempenho tenha melhorado, a nova varivel mantida. Caso contrrio, descartada. Estes passos so repetidos at esgotar as variveis, gerando um subconjunto final de variveis que sero usadas para treinar os modelos que avaliaremos no prximo passo. Um dos principais algoritmos de seleo de variveis o Boruta, que utiliza Random Forests para realizar a seleo.

3.7. Mdulo 07 - Treinamento de Algoritmos de Machine Learning

  • Selecionadas as variveis que utilizaremos para treinar os modelos, podemos, enfim, trein-los.

  • Para isso nosso conjunto de dados tem que estar dividido em teste e treino

  • O conjunto de treinamento ser utilizado para ensinar aos modelos o padro que queremos identificar e o rtulos (ou varivel alvo) que queremos predizer.

  • O conjunto de teste ser utilizado a posteriori para avaliar o aprendizado dos modelos.

  • H quatro tipos de modelos de Machine Learning:

    1. Aprendizado supervisionado: utilizado principalmente para classificao, regresso e sries temporais. Utilizado um conjunto de dados de treinamento que contm a varivel alvo, de modo que possamos supervisionar o aprendizado do modelo, explicitando seus erros e acertos.
    2. Aprendizado no-supervisionado: utilizado para problemas de clusterizao (agrupamento) em que no sabemos o resultado da varivel alvo ou sada, de modo que no conseguimos supervisionar o aprendizado do modelo.
    3. Aprendizado semi-supervisionado: uma mescla dos dois anteriores.
    4. Aprendizado por reforo: muito utilizado para resolver problemas em tempo real, uma vez que seu treinamento feito constantemente durante seu uso. muito utilizado para testes AB, por exemplo. Ele se baseia na ttrade cclica: Agente --> Ao --> Ambiente --> Recompensa --> Agente

  • Existem diversos modelos diferentes que sero estudados na disciplina de Fundamentos de Machine Learning.

  • Aqui utilizaremos cinco modelos:

    • Modelo de mdia: simplesmente a mdia por categoria e ser usado como linha de base para comparao com os demais modelos.
    • Modelo de Regresso Linear: simples e de fcil implementao. Permite avaliar a linearidade das variveis.
    • Modelo de Regresso Linear Regularizada (Lasso): regulariza as variveis dando diferentes pesos.
    • Modelo de Regresso Linear Regularizada (Ridge): regulariza as variveis dando diferentes pesos.
    • Modelo de Regresso por Random Forest:
    • Modelo de Regresso por XGBoost: usa mtodos de begging e boosting para selecionar as variveis. um modelo muito til e robusto.





  • Aps treinarmos os modelos, precisamos, antes de medir as mtricas de desempenho finais contra os dados de teste, realizar rodadas de cross-validation. Isto necessrio para garantir que no tenhamos, aleatoriamente, escolhido conjuntos de dados de teste e treino particularmente bons (ou ruins).

  • DVIDA: CROSS-VALIDATION FEITO PARA TODOS OS MODELOS OU APENAS PARA O QUE SELECIONARMOS COM BASE NAS MTRICAS DE DESEMPENHO CONTRA OS DADOS DE TESTE??---> R: Prioritariamente fazemos o cross-validation para todos os modelos implementados justamente a fim de identificar aquele que melhor desempenha sobre os dados.

    • A ideia do cross-validation dividir o conjunto de dados de treino em subconjuntos (pseudo-rplicas), retreinar os modelos e calcular seu desempenho diversas vezes, de modo que tenhamos uma distribuio de valores que possamos comparar com o que obtivermos com os dados de teste.
    • O cross-validation realizado de forma diferente para problemas de classificao e regresso e problemas de sries temporais. Isto porque sries temporais so cronologicamente contextualizadas e, portanto, criar as pseudorplicas de forma aleatria no faria sentido. Por isso, em problemas de sries temporais, as pseudorplicas so feitas sequencialmente com cortes sucessivos do conjunto de dados.

3.8. Mdulo 08 - Hyperparameter Fine-Tuning

  • O ajuste fino de hiperparmetros visa obter os melhores valores para os parmetros do modelo escolhido, fazendo com que sua performance melhore ainda mais. A melhora de performance costuma girar em torno de 2% a 3% do desempenho, por isso precisamos avaliar o custo-benefcio de sua aplicao.
  • Isto porque h trs mtodos principais de obteno dos valores dos parmetros:
    1. Random Search: arbitrariamente escolhemos um rol de valores que cada parmetro pode assumir e um nmero de combinaes que queremos testar, e o algoritmo de busca encontra a melhor combinao de valores dentre aqueles que estipulamos a priori no nmero de sorteios. bastante rpido, mas pouco eficaz, uma vez que pode acabar no encontrando os reais valores ideais dos parmetros (e nem chegando prximo deles).
    2. Grid Search: arbitrariamente escolhemos um rol de valores que cada parmetro pode assumir e o algoritmo avalia todas as combinaes possveis dos valores que estipulamos. Apesar de ser muito eficaz e, de fato, encontrar a melhor combinao de valores, pode demorar muito tempo no processo.
    3. Bayes Search: arbitrariamente escolhemos um rol de valores que cada parmetro pode assumir e o algoritmo usa o teorema de Bayes para encontrar a melhor combinao de valores. O teorema de Bayes usa conhecimento anterior (de testes realizados anteriormente com os valores do parmetro) para calcular a probabilidade posterior dos valores e achar a melhor estimvativa possvel. o mais eficiente dos trs por otimizar tempo e, ainda assim, ser satisfatrio em seus resultados.
  • Os algoritmos de fine-tuning podem demorar muitas horas para rodar pois a cada rodada de ajuste dos parmetros ele retreina o modelo e re-estima mtricas de desempenho para estipular o melhor conjunto de hiperparmetros.

3.9. Mdulo 09 - Interpretao e Traduo dos Erros

  • Os erros e mtricas de avaliao de aprendizado e desempenho dos modelos servem para que possamos aferir sua capacidade de generalizao e predio e para que possamos nos preparar para seu erro potencial.
  • Existem diversas mtricas diferentes que podem ser implementadas para os modelos, dependendo do modelo que estamos utilizando e do problema que estamos resolvendo.
  • Problemas de classificao usam mtricas como
    • Acurcia: quantas vezes o modelo acertou no geral
    • Retorno ou captura (recall): dos eventos de uma classe que tnhamos, quantos o modelo identificou corretamente
    • Preciso (precision): quantas vezes o modelo afirmou uma classe e ela era de fato aquela classe
    • F1-score: mdia harmnica entre retorno e preciso
  • Problemas de regresso usam mtricas como:
    • MAE (Mean Absolute Error):
    • MAPE (Mean Absolute Percentage Error):
    • MSE (Means Square Error):
    • RMSE (Root Mean Square Error):
    • ME (Mean Error):
  • Estas mtricas sero melhor estudadas na disciplina de Fundamentos de Machine Learning

3.10. Mdulo 10 - Implantao (Deploy) do Modelo escolhido em Produo

  • J temos o modelo selecionado treinado, ajustado e avaliado de modo que possamos, ento implement-lo e coloc-lo em produo. Isto significa disponibiliz-lo para ser consumido e utilizado.
  • Podemos disponibiliz-lo de diversas formas, como em aplicativos, dashboards, websites, etc. Geralmente isto feito atrav;s de uma API que intermediar as requisies realizadas pelo usurio e o modelo hospedado em nuvem.
  • Desta maneira devemos carregar o modelo, que ainda est em ambiente local, para ambiente de desenvolvimento em nuvem, onde ser testado e, por fim, para o ambiente em produo, onde ser empregado efetivamente.
  • Para isso precisamos criar um Handler API, que far a intermediao de requisies entre o usurio, o ambiente de produo e de volta para o usurio.
